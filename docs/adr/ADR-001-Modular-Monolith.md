# ADR-001 Modular Monolith Architecture

Status: Accepted

## Context

Need strong modularity without microservice complexity.

## Decision

Use Spring Modulith and Modular Monolith architecture.

## Alternatives Considered

- Traditional Monolith
- Microservices

## Consequences

### Positive
- Easier deployment
- Faster development
- Lower operational cost

### Negative
- Single deployment unit
